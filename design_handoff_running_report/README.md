# Handoff: Strava 月次ランニングレポート リデザイン (v2)

## Overview
既存の `strava-report` プロジェクト（Python で静的 HTML を生成する月次ランニングレポート）のリデザイン。
1本の長いスクロールページを **4タブ構成のモバイルファーストなダッシュボード** に再構成したもの。

- 対象: 個人用ダッシュボード（閲覧者は本人のみ、ローカル配信 `serve_report.py`）
- 目的: 情報設計の改善（長すぎるページの分割）、プランテーブルの可読性、スマホ表示

## About the Design Files
このフォルダの HTML は **デザインリファレンス（プロトタイプ）** であり、そのまま本番コードとして使うものではない。
実装タスクは、これらのデザインを **既存の `strava-report` コードベースの環境で再現すること**。
具体的には `report_html.py` が生成する HTML テンプレートを、このデザインに沿って書き換えるのが最も自然
（現行と同じく: 依存は CDN のみ・単一 HTML 出力・ビルド工程なし。Vanilla JS + inline `<script>` で十分。
フレームワーク導入は不要だが、導入するなら Preact 等の軽量なものに留めること）。

- `Running Report v2.dc.html` … デザイン本体（独自ランタイムのコンポーネント形式。マークアップとスタイル値の参照用）
- `reference_original_202607.html` … 現行の生成物（データ構造・全セクションの参照用）

## Fidelity
**High-fidelity**。色・タイポグラフィ・余白・角丸は最終値。ピクセル基準で再現すること。
ただしデータはすべて `report_html.py` のパイプライン（runs CSV / coach_cache JSON / pbs.json / races.json / Garmin データ）から動的に埋めること。デザイン内の数値は 2026年7月 の実データのサンプル。

## 全体構造
- 背景 `#F4F3F0`、コンテンツは `max-width: 760px` 中央寄せ、左右 padding 16px、カード間 gap 16px
- **スティッキーヘッダー**: `rgba(255,255,255,.94)` + `backdrop-filter: blur(8px)`、下線 `1px #E7E5E1`
  - 1段目: オレンジのバー(10×22px, #FC4C02, r3) + 「2026年7月」(19px/800) + 「ランニングレポート」(12px #78716C) + 右に月ナビボタン（← 6月 / 最新 ✦）
  - 2段目: タブ4つ（今日 / プラン / AIコーチ / 記録）。13.5px/700、アクティブは文字 `#1C1917` + 下線 `3px solid #FC4C02`、非アクティブ `#78716C`。横スクロール可 (`overflow-x:auto`)
- カード共通: 白背景、`border: 1px solid #E7E5E1`、`border-radius: 14px`、`padding: 18px`
- カード見出し(キッカー): 12px / 800 / `#78716C` / `letter-spacing: .08em`

## Interactions & Behavior
- タブ切替: クリックで表示セクション切替（SPA的に display 切替で可）。**選択タブは URL ハッシュ or localStorage に保持**し、リロードで復元
- 今週チェックボックス: トグル + localStorage 永続化（月別キー）
- アコーディオン（月間プラン週 / AIコーチセクション）: 単純トグル。▼/▲
- ホバー: ボタン類は現行同様 `transform: translateY(-1px)` 程度で控えめに
- 現行ヘッダーの機能ボタン（データ更新 / AI 評価 / Garmin 取得）と更新ステータスは**このモックには未収載**。実装時はヘッダー1段目の右側（月ナビの左）に同スタイルの小型ボタンとして残すこと

## Responsive
- モバイルファースト。単一カラムで 375px〜760px はそのまま成立
- 各行は `flex-wrap: wrap`（説明テキスト `min-width: 150〜160px` で折返し時に下段へ）
- ミニ統計 / PB は `auto-fit + minmax` グリッド
- `@media (min-width: 768px)` で任意の強化: 今日タブの「今日のメニュー+レースカウントダウン」を2カラム化してもよい
- タブバーは `overflow-x: auto`

## State Management
- `tab: 'today' | 'plan' | 'coach' | 'records'`（初期 'today'）
- `openWeek: number`（初期 = 当該週番号、0 で全閉）
- `openCoach: Record<string, boolean>`
- `done: Record<string, boolean>` → localStorage `rr2-done-<YYYYMM>`
- データはすべてビルド時（report_html.py）に埋め込み。fetch なし
